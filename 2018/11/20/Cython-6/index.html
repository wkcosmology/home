《Cython》第六章：组织你的Cython代码

Posted on 2018-11-20 | In Note , Cython
本文主要介绍了如何去组织Cython代码，并且和Python融洽地相容
  • 如果我们的程序代码全部在一个文件中，那我们就不需要本章的内容了。但是一般对于比较大的项目，我们总是需要把代码分布在不同的文件中，然后通过文件之间的导入和调用来组织我们的程序结构
  • 我们之前已经讲过如果从python程序中调用cython程序，但是原来的方法只能调用def和cpdef的程序
  • 我们这一章讲的东西主要有两个用途：
    • 一个cython文件如何调用另一个文件中的cdef，cpdef函数
    • Cython文件之间的调用，能够实现C层次的效率，避免Python函数的调用开销
  • Cython中引入了三种类型的文件
    • .pyx是实现文件
    • .pxd是声明或者定义文件，类似于头文件
    • .pxi是include文件
  • Cython中提供cimport声明提供在编译的时候在C层次上调用其他文件里面的东西

    和实现文件

  • 我们先说明一下实现和声明的区别

    • 一个函数或者类方法的声明指的是函数或者方法的签名：声明类型(cdef或者cpdef)；函数或者方法的名称；参数列表（不包括那个冒号）
    • 一个cdef class的声明包括：cdef class:和拓展类型的名称；所有的附属属性（成员变量）的名称；所有的类方法的声明
  • 我们归纳一下在pxd声明文件中的元素：
    • C类型声明：ctypedef, struct, union和enum
    • 外部C/C++库声明，即cdef extern
    • 模块级别的cdef和cpdef函数声明
    • cdef class拓展类型的声明
    • 拓展类型的cdef附属属性(attribute)声明
    • 拓展类型的cdef和cpdef方法声明
    • C级别的inline函数和类方法的实现
  • 再归纳一下在pyx实现文件中的元素：
    • Python函数或者非inline的C函数或者类方法的实现
    • 纯Python类的定义
    • 在IF或者DEF宏外的可执行python代码
  • 最后总结一下注意事项
    • pxd文件中的内容都是给其他的cython文件看的，不是给python文件看的，所以所有的def的不能在pxd文件中出现，否则会报错
    • 在pxd中声明的东西不能再在pyx中声明，包括struct, union, enum, 拓展类型的附属属性，但是在函数定义的时候也需要函数签名，这个不算重复声明
    • 利用pxd主要是为了其他的cython程序能够调用cdef和cpdef的函数或者struct等，并且调用cpdef的时候不会经历中间Python变量的转换，效率和底层C一样

cimport声明

  • 我们上面介绍了pyx和pxd文件的分离以方便其他的cython文件通过pxd文件在C级别上调用，那么这种调用也不是用import而是用cimport
  • import和cimport区别如下
    • cimportPython函数（def定义的函数）会引发编译错误；import仅仅在C中支持的声明也会引发编译错误
    • 我们可以import拓展cdef class或者cpdef的函数，但是更推荐使用cpimport。因为import导入的中间会使用Python对象来作为参数和返回值，但是cimport的情况下，会尽量使用C层次的形式，效率更高；除此之外，import导入的文件中的cdef和C级别定义的东西都不能调用

  • 声明文件中也允许使用cdef extern来包含C/C++的头文件

    这样我们就可以在其他的Cython文件中调用这些了

预定义声明文件

  • Cython中内置了一组声明文件，包含了C/C++, Python中常用的函数和结构体等
    • C对应的声明文件是libc：包含了C中的stdlib，stdio，math，string，stdint等头文件
    • Cpp对应的声明文件是libcpp：包含了Cpp中的标准库中的容器类：vector, list, string, map, pair, set等，还有算法库algorithm
    • Python对应的声明文件是cpython：包含了CPython源码中的C头文件，通过这个可以很容易使用Python/C API
