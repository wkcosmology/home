我的vim配置

Posted on 2019-06-03 | In Blog , ComputerScience
这篇文章主要介绍我是如何配置我的vim的，也是警戒自己以后不要再随便折腾vim了，现在已经很好了。

我的vim配置主要是基于Spacevim，加上一些调整。

Spacevim简单介绍

Spacevim是一个社区驱动的模块化Vim IDE，以模块的方式组织管理插件以及相关配置。
其主要思想有四点：

  1. 记忆辅助：所有快捷键，根据其功能的不同分为不同的组，以相应的按键作为前缀，例如 b 为 buffer 相关快捷键前缀，p 为 project 相关快捷键前缀， s 为 search 相关快捷键前缀，h 为 help 相关快捷键前缀。
  2. 可视化交互：创新的实时快捷键辅助系统，以及查询系统，方便快捷查询到可用的模块、插件以及其他更多信息。
  3. 一致性：相似的功能使用同样的快捷键，这在 SpaceVim中随处可见。这得益于明确的约定。其他模块的文档都以此为基础。
  4. 社区驱动：社区驱动，保证了 bug 修复的速度，以及新特性更新的速度。

我在Spacevim上的配置和改动

Spacevim在两个地方提供了可供用户自行配置：

toml个人配置

第一个添加个人配置的地方在：~/.Spacevim.d/init.toml

TOML是一种旨在成为一个小规模、易于使用的语义化的配置文件格式，它被设计为可以无二义性的转换为一个哈希表。
“TOML”这个名字是“Tom’s Obvious, Minimal Language（汤姆的浅显的、极简的语言）”的首字母略写词[2]。“Tom”指它的作者Tom Preston-Werner。
TOML的语法广泛地由key = “value”、[节名]与#注释构成。
它支持以下数据类型：字符串、整形、浮点型、布尔型、日期时间、数组和图表。

vim个人配置

第二个添加个人配置的地方在~/.Spacevim.d/autoload/myspacevim.vim

YCM配置

YCM(youcompleteme)是一个非常强大的vim自动补全插件。我们在spacevim中enable了YCM之后，需要到仓库位置进行编译，按照Spacevim默认的仓库位置，是在~/.cache/vimfiles/repos/github.com/Valloric/YouCompleteMe，现在推荐使用python进行编译，就是python3 install.py。
为了能在vim中自动补全众多第三方库，我折腾了一个周末，终于找到办法了。

  1. 添加Python解释器位置，比如我的在/Users/wangk/anaconda3/bin/python3.6,那么我们找到~/.ycm_extra_conf.py文件中interpreter_path位置，输入上面解释器的位置，并且要求解释器跟上面编译YCM的解释器是同一个；

  2. 添加python库的位置，在python中随便导入一个库，然后查看文件位置

    比如我的电脑显示/Users/wangk/anaconda3/lib/python3.6/site-packages/pandas/__init__.py，那么我的python库位置就是/Users/wangk/anaconda3/lib/python3.6/site-packages
    这时候我们需要在bash的配置文件中(一般是~/.bashrc，~/.bash_profile或者~/.zshrc)中加入export PYTHONPATH='/Users/wangk/anaconda3/lib/python3.6/site-packages/'。大功告成！

最后的话

Vim是一个经久不衰的编辑器，是否用Vim纯粹看个人爱好。vim的编辑方式毫无疑问是具有很高效率的，并且现在很多的IDE都支持vim键位，比如PyCharm，VDCode，所以即使不直接只用vim，我也建议了解一下vim键位。另外，使用vim或者emacs的一个好处就是可拓展性很强，甚至可以在比较小的时间付出下写一些自己需要的功能脚本。
